# Sales — Events

Subscribe to sales events in your module's `subscribers/` directory:

```typescript
// src/modules/<your_module>/subscribers/order-created.ts
export const metadata = {
  event: 'sales.order.created',
  persistent: true,
  id: 'your-module-order-created',
}

export default async function handler(payload, ctx) {
  // React to new orders
}
```

## Lifecycle events

These events fire during internal calculation pipelines. They are excluded from business rule triggers but can be subscribed to for custom pricing or tax logic.

| Event | Description |
|---|---|
| `sales.document.totals.calculated` | Document totals were recalculated |
| `sales.document.calculate.before` | Before document calculation pipeline |
| `sales.document.calculate.after` | After document calculation pipeline |
| `sales.line.calculate.before` | Before line calculation |
| `sales.line.calculate.after` | After line calculation |
| `sales.tax.calculate.before` | Before tax calculation |
| `sales.tax.calculate.after` | After tax calculation |
| `sales.shipping.adjustments.apply.before` | Before shipping adjustments are applied |
| `sales.shipping.adjustments.apply.after` | After shipping adjustments are applied |
| `sales.payment.adjustments.apply.before` | Before payment adjustments are applied |
| `sales.payment.adjustments.apply.after` | After payment adjustments are applied |
