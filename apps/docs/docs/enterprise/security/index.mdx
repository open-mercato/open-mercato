Open Mercato Enterprise Security brings personal account protection and operational security controls into one place.

It is built for teams that need stronger sign-in protection, clearer user guidance around account safety, and extra confirmation for actions with higher business risk.

The module focuses on three areas:

- hardened self-service password change
- MFA for both end users and administrators
- sudo re-authentication for sensitive actions

## In this section

- [Change password](./change-password) covers the user-facing password experience.
- [MFA for users](./mfa-user) covers setup, sign-in, and recovery from the user perspective.
- [MFA for admins](./mfa-admin) covers enforcement, rollout, compliance, and reset flows.
- [Extending MFA providers](./extending-mfa-providers) is the deep technical guide for adding another MFA method.
- [Sudo](./sudo) explains how sensitive actions are protected.

If you are documenting the product for users and operators, start with password, MFA, and sudo. If you are extending the module, use the MFA provider, sudo integration, and middleware guides.
