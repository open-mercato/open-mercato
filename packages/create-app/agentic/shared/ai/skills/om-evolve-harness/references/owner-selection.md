# Knowledge Owner Selection

Load this reference after the failing assertion is reduced.

| Failure | Smallest owner |
|---|---|
| Universal safety/writable-path invariant missing | Root `AGENTS.md` invariant. |
| Correct family not selected | Root Task Router row or skill description. |
| Conceptual cross-task contract wrong | One `.ai/guides/*.md` owner. |
| Branch procedure incomplete | One local skill reference. |
| Module ID/route/event/ACL/DI/spot fact wrong | Generated-facts extractor, not hand-written prose. |
| Exact installed detail unavailable | Context resolver/snapshot/package publishing. |
| Shared automation needs standalone delta | Narrow external skill override/config. |
| Skill dependency/layout missing | Tiers manifest/installer closure. |
| Tool edits wrong path or skips a guard | Tool hook/rule. |

Choose exactly one primary owner. Other files point to it. If the change needs two owners, split the assertion or justify why one contract spans both and add a consistency check.
