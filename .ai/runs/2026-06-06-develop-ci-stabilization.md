- Standalone CI run `27067843083` (PR head `05dfd2ed6`) still failed TC-CRM-059/060/061 "Status filter added via Add condition shows … tone in chip". Artifact `error-context.md` proved the advanced filter popover stayed OPEN with a fully-built rule (`Status / is / Active`) after the test's single `Escape`; the chip strip only renders when the popover is closed (`ActiveFilterChips` returns null while `popoverOpen`), so no chips appeared. Root cause: the fe790ef3f window-Escape listener gated on `event.defaultPrevented`, but Radix keeps a just-closed Select's dismissable layer mounted during its exit animation and that layer calls `preventDefault()` on the next Escape — so picking the value left a layer that swallowed the close Escape. On loaded standalone runners the `animationend`→unmount is delayed, widening the swallow window (passes on the lighter sharded ephemeral). Fix: the listener no longer trusts `defaultPrevented`; it defers Escape only when a dropdown is *actually* open (`[role="combobox"][aria-expanded="true"]`, which Radix flips synchronously on selection), and otherwise closes + flushes the panel.
