# Execution Plan: om-troubleshooter — propose fix and wait for confirmation

Tracking issue: https://github.com/open-mercato/open-mercato/issues/3002

## Goal

Add a "propose fix → wait for confirmation" gate to the `om-troubleshooter` skill so the agent describes the diagnosed root cause and proposed fix and waits for explicit user confirmation before applying any **mutating** change, while still running **read-only** diagnostics freely.

## Scope

- Edit `packages/create-app/agentic/shared/ai/skills/om-troubleshooter/SKILL.md` only.
- Add a confirmation-gate step to the Diagnostic Flow.
- Add a rule to the `## Rules` section.
- Reword `Fix:` lines to `Proposed fix:` for consistency with the gate.
- Add an explicit read-only vs. mutating classification.

## Non-goals

- No changes to the `dist/` copies — those are build artifacts regenerated by `build.mjs`.
- No changes to other skills (`om-help`, etc.) beyond the troubleshooter.
- No behavioral change to actual product code; this is agent-guidance documentation only.

## External References

None.

## Risks

- Over-prompting: if the gate does not exempt read-only diagnostics, the skill would ask before every `grep`/`typecheck`. Mitigated by an explicit read-only allowlist.
- Drift between source and generated `dist/` copies: acceptable — `dist/` is rebuilt from source, never hand-edited.
