# Step 2.3 ‚Äî Verification Notes

**Date:** 2026-04-18
**Step:** 2.3 ‚Äî Spec Phase 0 ‚Äî Restore loading of generated `ai-tools.generated.ts`
contributions in the runtime tool-loader.
**Branch:** `feat/ai-framework-unification`

## Scope

Wire module-contributed AI tools from `ai-tools.generated.ts` into the runtime
tool registry through the existing `registerMcpTool` / `mcp-tool-adapter.ts`
path. Code Mode tools (`search` + `execute`) remain loaded unchanged and are
additive.

## Files Touched

- `packages/ai-assistant/src/modules/ai_assistant/lib/tool-loader.ts`
  - Added exported `AiToolConfigEntry` type mirroring the generator contract.
  - Added `registerGeneratedAiToolEntries(entries)` ‚Äî validates shape,
    registers via `registerMcpTool`, idempotent, fail-soft on malformed
    entries.
  - Added `loadGeneratedModuleAiTools()` ‚Äî dynamic import of
    `@/.mercato/generated/ai-tools.generated`, returns 0 when the file is
    missing (tests / pre-generate).
  - `loadAllModuleTools()` now calls `loadGeneratedModuleAiTools()` after
    the Code Mode bootstrap.
- `packages/ai-assistant/src/modules/ai_assistant/lib/ai-tools-generated.d.ts`
  - `declare module '@/.mercato/generated/ai-tools.generated'` shim so the
    dynamic import type-checks inside the package without relying on the
    app's generated file.
- `packages/ai-assistant/src/modules/ai_assistant/lib/__tests__/tool-loader.test.ts`
  - 5 new unit tests covering: (a) modules with populated `ai-tools.ts`
    register, (b) modules without `ai-tools.ts` (empty/undefined tools)
    stay silent, (c) `mcp-tool-adapter.ts` resolves the registered tools
    with the original description + input schema + handler (shape parity),
    (d) `registerMcpTool` idempotency (re-run does not duplicate),
    (e) malformed tool objects are skipped with a warning.

## Risks & Mitigations

- **Duplicate registrations at boot**: `registerMcpTool` warns on
  overwrite. Confirmed idempotent via unit test.
- **Generated file missing at runtime**: `loadGeneratedModuleAiTools`
  catches the dynamic-import failure and logs a warning instead of
  crashing the MCP server.
- **Malformed module exports**: `isModuleAiTool` type guard rejects
  entries missing `name` / `description` / `inputSchema` / `handler`
  and logs a warning; registration continues for valid siblings.
