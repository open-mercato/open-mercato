## Overview

Dependabot PR [open-mercato/open-mercato#2016](https://github.com/open-mercato/open-mercato/pull/2016) targets `main`. The repository's working branch is `develop`, and per project policy dependency bumps must land on `develop` first. This run ports that single-file lockfile change onto a fresh `fix/bump-ws-8-20-1` branch off `develop` and opens a new PR. After merging, the original PR will be closed as superseded.

The upstream change addresses a security advisory in `ws`: `websocket.close()` could disclose uninitialized memory when called with a `TypedArray` (e.g. `Float32Array`) as the `reason` argument. Fixed in `ws@8.20.1` (commit `c0327ec1`). `ws` is a transitive dependency in this repo (no direct `package.json` entry); the affected resolution is `"ws@npm:^8.17.1, ^8.18.0"` â†’ currently `8.19.0` on develop.

## Goal

- Bump the `^8.17.1, ^8.18.0` `ws` resolution in `yarn.lock` from `8.19.0` to `8.20.1` on develop, so develop carries the same security fix proposed in PR #2016.

## Risks

- **Lockfile drift**: yarn 4 (Berry) computes the checksum locally from the tarball. The checksum in the upstream PR matches the npm registry's published tarball; running `yarn install` locally MUST regenerate the same checksum. If it doesn't, regenerate via `yarn install` and re-verify. Mitigation: `yarn install --immutable` after the patch.
- **Transitive consumer breakage**: anything importing `ws` through the `^8.17.1`/`^8.18.0` ranges (likely `webpack-dev-server`, `jest-environment-jsdom`/`puppeteer`/etc.) gets the new minor. The diff between 8.19.0 and 8.20.1 is one new export (`PerMessageDeflate`), one new option (`closeTimeout`), and the security fix â€” all additive. Mitigation: the full validation gate (build:packages, typecheck, test, build:app) must pass.
- **No source-level tests**: nothing in this repo exercises `ws.close()` directly. The validation gate substitutes for a unit test here; this is consistent with how the codebase handles dependency bumps (per `.ai/runs/2026-05-20-migrate-dependabot-deps-to-develop.md`).
