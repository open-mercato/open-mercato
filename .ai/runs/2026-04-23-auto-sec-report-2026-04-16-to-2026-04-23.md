- Goal: Produce an aggregate OWASP-oriented security report for the newest merged pull requests into `develop` from 2026-04-16 through 2026-04-23 UTC, ship the report as a docs-only PR, and preserve resumability through the execution plan.
- Window: 2026-04-16 through 2026-04-23 UTC.

## Scope

- Create the execution plan and checkpoint branch in an isolated worktree.
- Run `auto-sec-report-pr` in fragment mode for queued PR units.
- Aggregate the fragments into markdown and HTML artifacts under `.ai/analysis/`.
- Open a docs-only PR against `develop` with the required labels and summary.

## Non-goals

- Analyze the 29 truncated units in this invocation.
- Change product code, tests, or implementation behavior.
- Merge the delivery PR.

## Risks

- The queue is large enough that the run may need to stop in-progress and hand off to `/auto-continue-pr`.
- Per-unit security review is heuristic and diff-based; findings must be confirmed by a human before action.
- PR claim locks may cause some units to be skipped if another actor owns them.

### Phase 1: Queue and plan

1. Resolve the merged PR queue for the requested UTC window and record the truncation boundary.
2. Create the isolated worktree branch and commit this execution plan as the run anchor.

### Phase 2: Per-unit analysis

### Phase 3: Aggregation

1. Concatenate the completed fragments into the aggregate markdown report.
2. Build the consolidated "Next steps — go deeper" list and heatmap summary.
3. Render the HTML mirror and perform docs-only validation.

### Phase 4: PR delivery

1. Open the docs-only PR against `develop`.
2. Apply `review`, `documentation`, `security`, and `skip-qa` labels with explanatory comments.
3. Run the post-open review flow and leave the comprehensive summary comment.
